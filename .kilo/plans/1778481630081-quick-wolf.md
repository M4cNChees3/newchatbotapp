# Plan: Create New Supabase Database and Integrate with Netlify

## Objective
Create a new Supabase project to replace the expired database, update the application configuration, and ensure Netlify deployment uses the new Supabase credentials.

## Steps

### 1. Create New Supabase Project
   - Go to https://supabase.com/dashboard and create a new project.
   - Note the Project Reference (e.g., `xxxxx.supabase.co`).
   - Obtain the `anon` public key and `service_role` secret key from Settings > API.

### 2. Reset and Initialize Database
   - Option A: Use Supabase CLI to reset local database and apply migrations.
     ```bash
     supabase db reset  # Linked to new project via supabase link
     supabase migration up
     ```
   - Option B: Use Supabase Dashboard SQL editor to run all migration files in order:
     1. `supabase/migrations/20251123170752_create_initial_schema.sql`
     2. `supabase/migrations/20251124053643_fix_security_issues.sql`
     3. `supabase/migrations/20251124053833_complete_security_fixes.sql`
     4. `supabase/migrations/20251124121947_fix_athletes_insert_policy.sql`
     5. `supabase/migrations/20251127030703_add_admin_role_system.sql`
     6. `supabase/migrations/20251209000000_create_admin_user.sql`

### 3. Update Frontend Environment Variables
   - Edit `.env` file (local) with new values:
     ```
     VITE_SUPABASE_URL=https://<new-project-ref>.supabase.co
     VITE_SUPABASE_ANON_KEY=<new-anon-key>
     ```
   - In Netlify dashboard:
     - Go to Site Settings > Environment variables.
     - Update (or add) `VITE_SUPABASE_URL` and `VITE_SUPABASE_ANON_KEY` with the new values.
     - Ensure these are set for production context.

### 4. Configure Supabase CLI (Optional but Recommended)
   - Link CLI to new project:
     ```bash
     supabase link --project-ref <new-project-ref>
     ```
   - Pull down any remote changes if needed:
     ```bash
     supabase db pull
     ```

### 5. Deploy/Update Supabase Edge Functions
   - The project includes two edge functions: `chat` and `process-document`.
   - Deploy functions to the new project:
     ```bash
     supabase functions deploy chat --project-ref <new-project-ref>
     supabase functions deploy process-document --project-ref <new-project-ref>
     ```
   - Set required secrets for each function in the new project:
     - For `chat` function:
       - `GROQ_API_KEY` (from Groq service)
       - `SUPABASE_URL` (new project URL)
       - `SUPABASE_SERVICE_ROLE_KEY` (new project service_role key)
     - For `process-document` function:
       - Same as above if it uses Supabase client.
   - Set secrets via dashboard or CLI:
     ```bash
     supabase secrets set --env-file .env.supabase  # If you have an env file
     # Or individually:
     supabase secrets set GROQ_API_KEY=your_key
     supabase secrets set SUPABASE_URL=https://<new-project-ref>.supabase.co
     supabase secrets set SUPABASE_SERVICE_ROLE_KEY=<service_role_key>
     ```

### 6. Verify and Test
   - Run the application locally to ensure it connects to the new Supabase database.
   - Test authentication, data read/write, and function invocations.
   - Deploy to Netlify (if not already set up for auto-deploy) and test production endpoints.
   - Check Netlify logs for any environment variable or connection issues.

### 7. Cleanup (Optional)
   - After verifying the new project works, consider deleting the old expired Supabase project to avoid confusion.

## Notes
- Ensure the `.env` file is not committed to git (it is already in `.gitignore`).
- The Supabase functions currently have `verify_jwt = false` in config.toml; adjust if authentication is required.
- The `chat` function now includes authentication middleware (added in a prior fix) which expects a valid user token; ensure frontend sends the user's Supabase JWT in the Authorization header.

## Estimated Time
- 30-60 minutes depending on familiarity with Supabase and Netlify.
